# Whisper Studio

Локальная обёртка для транскрипции аудио на базе **faster-whisper**.
Своё веб-приложение в стиле онлайн-сервисов, но всё работает на вашем
компьютере: без подписки, без лимитов, без отправки аудио на чужой сервер.

## Возможности

- Загрузка аудио/видео перетаскиванием + запись с микрофона
- Модели на выбор: `small`, `medium`, `large-v3` (для русского — `medium` или `large-v3`)
- Автоматическое использование GPU NVIDIA, если он есть (иначе CPU)
- Пакетная обработка нескольких файлов
- Просмотр с таймкодами, перемотка аудио по клику на время
- Редактирование текста прямо в браузере, переименование говорящих
- Экспорт в **TXT, DOCX, SRT, VTT**
- Локальная история всех транскрипций
- Светлая / тёмная тема
- Опционально — распознавание говорящих (Speaker 1 / Speaker 2)

## Установка

Нужен **Python 3.9+** ([python.org](https://www.python.org/downloads/) — при установке
на Windows отметьте галочку «Add Python to PATH»).

### Быстрый запуск

- **Windows:** двойной клик по `start.bat`
- **macOS / Linux:** `chmod +x start.sh && ./start.sh`

Скрипт сам поставит зависимости и откроет приложение в браузере.

### Запуск вручную

```bash
pip install -r requirements.txt
python app.py
```

Приложение откроется на `http://127.0.0.1:8756`.

> При первом запуске выбранная модель Whisper скачается автоматически
> (нужен интернет один раз, ~0.5–3 ГБ). Дальше всё работает офлайн.

## GPU-ускорение (необязательно, но сильно ускоряет)

`faster-whisper` использует видеокарту NVIDIA автоматически, если в системе
установлены **CUDA 12** и **cuDNN 9**. Без них приложение работает на процессоре —
медленнее, но полностью рабочее. Текущий режим показан внизу боковой панели
и на вкладке «Настройки».

## Говорящие (диаризация) — опционально

Чтобы появилась разметка «Speaker 1 / Speaker 2»:

1. Установите пакет:
   ```bash
   pip install pyannote.audio==3.3.2
   ```
2. Зарегистрируйтесь на [huggingface.co](https://huggingface.co) и примите условия
   модели `pyannote/speaker-diarization-3.1` на её странице.
3. Создайте токен доступа (Settings → Access Tokens) и задайте его перед запуском:
   - Windows: `set HF_TOKEN=ваш_токен`
   - macOS/Linux: `export HF_TOKEN=ваш_токен`

После этого в интерфейсе заработает переключатель «Определять говорящих».

## Где хранятся данные

Всё лежит рядом с программой в папке `data/`:
- `data/uploads/` — исходные аудиофайлы
- `data/recordings/` — транскрипции в формате JSON

Чтобы перенести или сделать резервную копию — просто скопируйте папку `data/`.

## Настройка

В начале файла `app.py` можно изменить:
- `DEFAULT_MODEL` — модель по умолчанию
- `PORT` — порт (по умолчанию 8756)

## Если что-то не работает

- **«python не найден»** — Python не установлен или не добавлен в PATH.
- **Долгая первая транскрипция** — идёт скачивание модели, далее будет быстрее.
- **Медленно на CPU** — это нормально: час аудио ≈ 1–3 часа обработки на `medium`.
  Помогает видеокарта NVIDIA или модель поменьше.
- **Ошибка про cuDNN/CUDA** — GPU-библиотеки не установлены; приложение всё равно
  работает на CPU.
